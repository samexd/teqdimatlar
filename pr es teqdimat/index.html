1. Giriş: Xətaların İdarə Edilməsi

Məqsəd qəfil çökmələrin (crash) qarşısını almaq və proqramı dayanıqlı etməkdir.

  • Proqramın axınını qorumaq.
  • Resurs sızmasının qarşısını almaq.
  • İstifadəçiyə aydın mesaj vermək.
// Təhlükəli kod nümunəsi
int total = 100;
int user_count = 0;

// CRASH! Proqram burada dayanır:
int result = total / user_count;

2. Xətaların Növləri

  • Compile-time: Kodun qrammatik səhvləri.
  • Runtime: İşləmə zamanı yaranan real problemlər.
  • Logical: Proqram işləyir, amma nəticə səhvdir.
// 1. Sintaksis (; yoxdur)
int x = 5

// 2. Runtime (Null pointer)
int* ptr = nullptr;
cout << *ptr; // Xəta!

3. Error Handling Anlayışı

Xətanı necə qarşılayırıq?

  • Ənənəvi (C Stili): Funksiyalar xəta kodu (-1) qaytarır.
  • Müasir (C++ Exceptions): Xəta baş verəndə kodun normal axını dayanır və "catch" blokuna keçir.
// C++ Stili
try {
  process_data();
} catch (...) {
  save_logs();
}

4. Exceptions Nədir?

Exception – proqramın normal gedişatını pozan bir obyektdir.

Xəta baş verdikdə, C++ onu tutacaq (catch) bir yer axtararaq funksiyalardan çıxır.

void check_age(int age) {
  if (age < 18) {
    throw std::runtime_error("Yetkin deyilsiniz!");
  }
}

5. try-blok

Xəta ehtimalı olan kodları `try` blokuna yazırıq.

Əgər xəta baş verərsə, blokun qalan hissəsi icra edilmir.

try {
  open_database();
  process(); // Xəta burada ola bilər
  close(); // Bura çatmaya bilər
}

6. catch-blok

`catch` bloku atılan xətanı tutur.

  • Mesajı ekrana çıxarır.
  • Proqramı təhlükəsiz bağlayır.
catch (const std::exception& e) {
  cerr << "Xəta: " << e.what();
  cleanup();
}

7. throw operatoru

Xəta vəziyyətini bildirmək üçün istifadə olunur.

İstənilən tip (int, string, class) atıla bilər.

if (file_missing) {
  // Kod atmaq
  throw 404;

  // Mətn atmaq
  throw "Fayl yoxdur!";
}

8. Standard Exceptions

C++ `` kitabxanası hazır siniflər verir.

  • std::out_of_range: Massiv sərhəddi.
  • std::invalid_argument: Yanlış parametr.
vector<int> v(5);
try {
  v.at(10) = 1; // Sərhəd xətası
} catch (const out_of_range& e) {
  cout << e.what();
}

9. Custom Exceptions

Standart xətalar yetmədikdə, öz xəta sinifimizi yaradırıq.

`std::exception` sinifindən miras almaq ən yaxşı yoldur.

class DBError : public std::exception {
public:
  const char* what() const noexcept {
    return "DB Xətası!";
  }
};

10. Exception Hierarchy

Bütün xətaların "atası" std::exception-dır.

Polimorfizm sayəsində Base class ilə bütün xətaları tuta bilərik.

try {
  // Kod...
} catch (const std::exception& e) {
  // Bütün standart xətaları tutur
  cout << e.what();
}

11. Exception Safety

Xəta zamanı proqramın davranışı:

  1. Basic Guarantee: Yaddaş sızması yoxdur.
  2. Strong Guarantee: Əməliyyat ya tam olur, ya heç olmur.
  3. No-throw: Xəta atmama zəmanəti.
// noexcept açar sözü
void funksiya() noexcept {
  // Xəta olsa terminate() olur
}

12. Stack Unwinding

Xəta atılanda lokal obyektlərin destruktorları avtomatik çağırılır.

Bu, yaddaş sızmasının qarşısını alır.

void process() {
  File f("log.txt");
  throw 1; // Xəta atıldı
  // f destruktoru çağırılır
}

13. Multiple Catch

Qayda: Xüsusi xətalar (derived) əvvəl, ümumi xətalar (base) sonda gəlməlidir.

try { ... }
catch (const out_of_range& e) {
  // Konkret xəta
} catch (const exception& e) {
  // Ümumi xətalar
}

14. Re-throwing

Xətanı tutub sadəcə qeydə alırıqsa, onu yenidən yuxarı ata bilərik.

catch (const exception& e) {
  log(e.what());
  throw; // Yenidən at
}
